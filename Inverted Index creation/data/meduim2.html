Windows 11 is Beautiful, Because Microsoft is Finally Sweating the Details

With its newest redesign of Windows, Microsoft wants to win back developers.

Owen Williams

2 days ago·5 min read

For the last few years, Microsoft has been on a comeback. The company has captured the developer audience by acquiring GitHub and reviving it, as well as building the world’s most popular code editor, VS Code, and bringing Linux to Windows 10.

Despite that success capturing the developer market, however, Microsoft struggled to find traction that got them to embrace building great new native apps for Windows. While Windows 10 was a huge upgrade that got better every year, it was riddled with inconsistencies, visually left a lot to desire, and had a messy story for those that did want to build apps.

Windows 11, which will be released later this year, is poised to change that with a total visual overhaul to the operating system, and the way apps are built. It’s a vast redesign that heavily uses transparent, glassy elements, moves the start menu to the middle of the taskbar, moves away from the animated ‘live tiles,’ and introduces well-considered, new design patterns for the first time in years.

But, what’s striking about Windows 11 is that it’s extensive, reaching to the wider operating system as a whole — instead of some corners receiving design updates, and others still looking like they came straight out of Windows 2000. For the first time since Windows Vista, Microsoft has taken the time to focus on bringing its new visual design language everywhere, and made it easier for developers to take advantage of it.

While Windows 10, which was first released in 2015 also came with a visual refresh dubbed Modern UI, which was an extension of the colorful, blocky Metro design language introduced in Windows 8. It was a promising improvement, but the problems were obvious from the start: Microsoft adopted its design language inconsistently between its own apps, and even built-in apps, which should have been a demonstration of how to use Modern UI well. Worse still, few developers properly implemented the new design language, if at all.

The most obvious place in Windows 10 where this friction showed was the Settings app. Not only was it clunky and poorly organized, it was an inconsistent mess, with buttons that looked like text labels, and every page using different elements in unpredictable ways. Worse still, it frequently kicked you out into the old-school Control Panel app because some features hadn’t been updated yet, making for a frustrating experience.

Using the Windows 11 beta, which is now available for testing, it feels like Microsoft took the time to sweat the details. That same Settings app has been rebuilt from the ground up and it’s immediately apparent the effort that’s gone into getting it right. Not only is it well organized, each page is consistent, uses the same design elements, and it doesn’t kick you out to some obscure part of the OS to perform a task anymore.

That same attention to detail is everywhere, from the new widgets panel that slides over from the left, to the banal places like the notification center and right-click menus, that have been fully redesigned as well. It works together well as a whole, rather than feeling dusty in corners — and it’s only in a preview stage right now.

It’s made available the same design elements it uses in its own apps to developers. New components such as Mica, for example, can be used by any app to bring a glassy, ‘dynamic’ background and title bar, allowing the user’s background to peek through an app and look at home on Windows 11, despite being built by a third-party.

While Microsoft made components like this available in Windows 10’s design language as well, it initially only allowed developers to use them when they built apps based on the company’s newer app framework, Universal Windows Platform (UWP). Microsoft really wanted developers to migrate to UWP, but it was an uphill battle because they would often need to rebuild their existing apps based on older frameworks like Win32 or .NET entirely.

UWP apps, from a user point of view were better for a number of reasons, including that they were easier to install and uninstall from the Windows Store, could better adapt to high-density displays, and felt more modern, but few developers were willing to put in the work to totally rebuild their apps — leaving the visual design feeling inconsistent between every individual app.

Worse still, only apps built with these modern frameworks could be listed in the store, meaning it was unpredictable what you could actually find there. Spotify was in the Windows Store, but Steam and Adobe Acrobat weren’t.

Windows 11 fixes this with two simple, but massive changes. First, developers can now use any of the new visual elements, like Mica and Acrylic, regardless of what framework they built their apps with. That means existing developers can adopt the fresh, new design, without totally rewriting their code — making them far more likely to do so.

Second, developers can now list their apps, regardless of how they’re built, in the Windows Store for the first time. That means far more apps will be listed in the store, a process that has already begun since the moment the floodgates opened, with streaming software OBS and Adobe Acrobat already being added in the last week.

Microsoft has also stopped taking its revenue cut from developers that list in the store and charge in-app purchases with their own payment engine instead of the company’s tools. That means marketplaces like Steam or Epic Games could come to the store, without losing a large chunk of their revenue to fees.

None of this is to say that Windows 10 was bad — it was improved leaps and bounds every year, and is a solid operating system, but it was deeply inconsistent, and had a confusing developer ecosystem that left it feeling fragmented despite the company’s efforts.

Windows 11 is a new chance at a beautifully designed, consistent operating system that’s embraced by both Microsoft’s own apps and the developer ecosystem as a whole. By taking the time to sweat the details in Windows 11 and get its own apps right, Microsoft has demonstrated how thoughtful and lovely apps can be on the operating system, and it’s made those tools available to developers properly for the first time.

The open question will be whether or not developers build new apps for the platform, sparking a renaissance, and existing developers embrace the new visual style. After years of languishing, Microsoft has a good shot at winning back the desktop —and developer attention — if it keeps it up.
